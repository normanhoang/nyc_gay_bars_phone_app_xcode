# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

Native **Swift/SwiftUI** iOS app — a tracker for NYC gay bars (browse list/map, log drinks today or backdated, view history/stats/badges). Single-user, fully local (UserDefaults), no network at runtime. Ported from an Expo/React Native app (the `legacy-expo/` archive referenced below is not currently in the tree).

- **iOS 26 only.** Uses SwiftUI's native Liquid Glass (`.glassEffect`). Requires Xcode 26+.
- The `.xcodeproj` is **generated by [xcodegen](https://github.com/yonyz/XcodeGen)** from `project.yml` — do not hand-edit it. After adding/removing/renaming source files, run `xcodegen generate` (sources are globbed from the `NYCGayBars/` dir).

## Commands

```bash
xcodegen generate                                                    # regenerate NYCGayBars.xcodeproj after file changes
xcodebuild -scheme NYCGayBars -destination 'platform=iOS Simulator,name=iPhone 17,OS=26.5' build
xcodebuild -scheme NYCGayBars -destination 'platform=iOS Simulator,name=iPhone 17,OS=26.5' test
# single test:
xcodebuild -scheme NYCGayBars -destination 'platform=iOS Simulator,name=iPhone 17,OS=26.5' test -only-testing:NYCGayBarsTests/GeoTests/testKnownZip
```

Device install (signing team must be set — `DEVELOPMENT_TEAM` is blank in `project.yml`; pass it on the command line or set in Xcode):

```bash
xcodebuild -scheme NYCGayBars -destination 'platform=iOS,id=<DEVICE_UDID>' -allowProvisioningUpdates DEVELOPMENT_TEAM=<TEAM> CODE_SIGN_STYLE=Automatic build
xcrun devicectl device install app --device <DEVICE_UDID> <built>.app   # retry once; first attempt often fails "connection interrupted"
xcrun devicectl device process launch --device <DEVICE_UDID> com.normanhoang.nycgaybars   # fails if phone is locked
```

App Store IPA (bump `CURRENT_PROJECT_VERSION` in `project.yml` first — ASC rejects duplicate build numbers — then `xcodegen generate`):

```bash
xcodebuild -scheme NYCGayBars -destination 'generic/platform=iOS' -archivePath build/NYCGayBars.xcarchive archive -allowProvisioningUpdates
xcodebuild -exportArchive -archivePath build/NYCGayBars.xcarchive -exportPath build/ipa -exportOptionsPlist build/ExportOptions.plist -allowProvisioningUpdates
# build/ is gitignored — if build/ExportOptions.plist is missing, recreate it:
# a plist dict with method=app-store-connect, teamID=DZRPJF9JB6, signingStyle=automatic, destination=export
```

There is no SwiftUI Preview-based screenshot harness. To screenshot a specific tab on the simulator, temporarily seed `ProcessInfo` env reads (e.g. a `START_PAGE` page nudge in `RootTabView`), launch with `SIMCTL_CHILD_START_PAGE=N xcrun simctl launch ...`, capture with `xcrun simctl io booted screenshot`, then **revert the scaffolding**. The MapKit view and a stuck system location alert can block screenshots — boot the sim fresh and `xcrun simctl privacy booted grant location-always com.normanhoang.nycgaybars` first.

To *interact* with the UI (taps/swipes/scrolls), host-level clicking is unreliable (no idb/axe installed; the Simulator window is often offscreen so cliclick can't reach it). What works: add a **temporary `bundle.ui-testing` target** to `project.yml` + an XCUITest that drives the flow and attaches screenshots (`XCTAttachment(screenshot:)`, `.keepAlways`), run with `-only-testing:`, extract via `xcrun xcresulttool export attachments --path <xcresult> --output-path <dir>` (its manifest.json maps exported names), then **revert project.yml, delete the test dir, rerun xcodegen**. XCUITest selector gotchas: `staticTexts["Stats"]/["History"]` also match the tab-bar labels, and stat-card subtitles shadow neighborhood names (FAVORITE BAR shows "Hell's Kitchen") — assert on unique markers like `"TOTALS"` or the calendar month title instead. Seed state before launch: `xcrun simctl spawn booted defaults write com.normanhoang.nycgaybars "@gaybars/visits" -data <hex of JSON [Visit]>` (same for `@gaybars/visited`, a JSON `[String]`).

SourceKit IDE diagnostics routinely flag cross-file types as "Cannot find X in scope" in this repo — stale-index noise; trust `xcodebuild` output.

## Architecture

Plain SwiftUI + two `ObservableObject` stores injected as environment objects from `NYCGayBarsApp` (`VisitsStore` → `BadgesStore`). No third-party deps.

### Data is generated, not hand-written
`NYCGayBars/Data/Resources/{bars,neighborhoods,zips,meta}.json` are **build artifacts**, decoded once by `AppData` into `AppData.bars/neighborhoodPolygons/zipCentroids/neighborhoods/region`. CLAUDE history references a `legacy-expo/` pipeline (`gay_bars.csv` + `scripts/*.mjs`) as the original source of truth — **that dir is not in the working tree**, so these JSON files are currently edited/regenerated **directly** with throwaway Python scripts.

Neighborhood polygons are built from external NYC neighborhood GeoJSON, not drawn by hand:
- **Outer boroughs (Brooklyn/Queens)** use the **Pediacities** common-name set (`custom-pedia-cities-nyc-Mar2018.geojson`); **Manhattan** polygons are hand-tuned (12th-Ave west edges; the HK/Chelsea divider follows W 37th St's diagonal).
- Each bar's `neighborhood` is assigned by **point-in-polygon**; a few border bars were reassigned by explicit user calls (e.g. BASEMENT NY → Williamsburg even though its pin falls just outside that polygon — `bar.neighborhood` drives filtering/badges, polygon containment only drives map tap).
- **Invariants:** every polygon must be **valid + simple + non-overlapping**, stored as a single ring. The reliable pipeline: `buffer(0)`-clean each input → `unary_union`/`difference` → `simplify(~0.0003, preserve_topology=True)` → `set_precision(1e-6)` (grid-snap) → take the largest exterior ring. Skipping the input clean or running `simplify` last on an invalid ring leaves **zero-width antennas that render as stray lines** inside the boundary. `shapely` is required (`pip install shapely`).
- When the neighborhood **set** changes, also update `meta.json`'s `neighborhoods` list, the affected `bar.neighborhood` values, and `Stats.borough()`'s `brooklynHoods`/`queensHoods` sets (a hardcoded name→borough map; stale names silently count as Manhattan and break the boroughs badge). Zip expectations in `LogicTests` are nearest-*bar* based.

### State + persistence
- `VisitsStore` — `visits`, `visitedBars`, `hydrated`; persists JSON to UserDefaults keys `@gaybars/visits`, `@gaybars/visited`. A `Visit` aggregates drinks per type for one local calendar day; a **zero-drink check-in** is a valid visit (`setVisited`). Backdated visits are stamped **noon local** so the ISO date round-trips the day key; future days are refused.
- `BadgesStore` — recomputes badges from visits, persists first-earned timestamps (`@gaybars/badgeDates`), and drives the unlock-toast queue. It is **not** self-subscribing: `RootTabView` calls `badges.reconcile(visits:visitedIds:)` on appear and on `visits.visits` / `visits.visitedBars` changes.

### `DayKey` (gotcha)
Day keys are `"year-month-day"` with a **0-indexed month** (mirrors JS `getMonth()`), so June 24 2026 is `"2026-5-24"`. `DayKey.toDate` rebuilds noon-local. All streak/badge/calendar logic depends on this round-trip — don't "fix" the month to 1-indexed.

### Logic lives in `Logic/` (pure, unit-tested)
`Stats` (all 30 badges + `MILESTONE_BADGE_IDS` + totals/streaks), `Geo` (distance, proximity ordering, ZIP lookup), `DayKey`, `Drinks`, `ZipQuery`. `NYCGayBarsTests` covers these.

### Liquid Glass (`Theme/Glass.swift`) — chrome vs panels
`.glassSurface(radius:bordered:)` (native `.glassEffect`) is for **interactive chrome only** (search bars, toggles, tab bar, buttons, toasts). Static content (stat cards, calendar, visit cards, badge tiles) uses a plain `.contentPanel()` / `Color.white.opacity(0.05)` rounded rect — native glass draws a luminous rim that reads as a border. Never put an `opacity` modifier on a glass element. `AppBackground` (gradient) must sit behind glass so it has colour to refract.

### Navigation shell (`Screens/RootTabView.swift`)
Three pages in a horizontal paging `ScrollView` (`.scrollTargetBehavior(.paging)` + `scrollPosition(id: $page)`) with a custom floating glass tab bar overlaid (content-hugging, fixed-width tabs, centered — not full-bleed). The active tab's magenta `Capsule` lozenge is placed by **`matchedGeometryEffect`** onto each tab's fixed-size content box (identical at every position — do not go back to manual offset math). **`pillPage` is decoupled from `page`:** a tap sets both, moving the pill with a fast `.snappy` (instant feel, no bounce); a swipe settles `page` → `.onChange(of: page)` springs `pillPage` with the bouncy `Anim.tab`. Wrapping the tap pill-move in *some* animation is required — without a transaction the matched-geometry pill is removed+reinserted and flashes. Explore's map mode disables horizontal paging via `TabSwipe` (the map needs to pan); `scrollDisabled` only applies on page 0. `TabSwipe` also broadcasts the current `page`; each page owns a `ScrollPosition` and scrolls itself to top when `tabSwipe.page` changes to another index — the reset happens while offscreen, so returning to a page always lands at the top with no visible jump.

### Map framing (`Screens/BarMapView.swift`)
Selecting a neighborhood programmatically reframes the camera; zooming out far enough (2+ neighborhoods visible, span > 1.2× framed) snaps the filter back to `All`. The framed span must be the **actual rendered** span, captured from the first `onMapCameraChange` after a reframe via the `awaitingFrame` flag — MapKit enlarges a region to fit wide/non-square bars, so using the *requested* `latitudeDelta` made wide neighborhoods (e.g. East Williamsburg) instantly snap back to `All`.

### Motion (`Components/Motion.swift`)
Reusable springs in `Anim` (chip/press/progress/toast/tab) mapped from the original RN reanimated values. `PressableScale` is a `ButtonStyle`; `CountUp`/`ProgressBar`/`Confetti`/`BadgeToast` are the animated primitives.

### Modals
`BarDetailSheet` (drink logging) is shown as a `.sheet` from Explore (logs today) and **in-place** inside `LogDayPicker` for backdated logging — when hosted in-place it takes an `onClose` closure instead of `@Environment(\.dismiss)`.
